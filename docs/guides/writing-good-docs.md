# 좋은 문서를 쓰는 방법

## 원칙
- 독자의 문제를 먼저 설명하고 해결책을 제시합니다.
- 예제는 복사/붙여넣기만으로 동작해야 합니다.
- 기본 → 고급 순으로 점진적 예시를 제공합니다.
- 경계조건과 실패 사례를 함께 명시합니다.

## 체크리스트
- 무엇을 하는가? 왜 필요한가?
- 입력(매개변수), 출력(반환값), 부작용
- 에러 시나리오와 복구 방법
- 성능/보안 주의사항
- 샌드박스 예제 링크

## 예시 구조
```md
# {API 이름}
- 개요
- 시그니처
- 매개변수/반환
- 예제(기본/고급)
- 사용 지침(제약/에러/성능/보안)
- 변경 이력
```
